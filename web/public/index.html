1 · Use with Claude Code

The reason pergam exists. Drop the skill once, then ask Claude to "armá un grid HTML de X" inside any project. The skill writes the HTML, posts it here, and returns a shareable URL — no manual upload, no copy-paste, no leaving the terminal.

git clone https://github.com/diesilveira/pergam
cp -r pergam/skills/post-html-grid ~/.claude/skills/

# Public 72h share (free, rate-limited, ephemeral):
export PERGAM_URL=https://api.pergam.dev

# OR self-hosted (private, durable, versioned — see option 2):
export PERGAM_URL=http://localhost:1111

Every future "publish this as a grid" in Claude Code lands here automatically, with the right metadata (title, type, your email as author). Iterating? Same id, new version — full history.

Designed for PR review boards, status dashboards, spec docs with Mermaid, research notes — anything your AI generates as a self-contained HTML report.

2 · Self-host (teams & durability)

One docker compose up and you have your own pergam with Postgres, versioning, immutable history, and zero TTL. Behind your VPN, on your internal domain, owned by your team.

git clone https://github.com/diesilveira/pergam
cd pergam
docker compose up -d
open http://localhost:1111

What you get over the public service:

  • No expiration — grids live forever in your Postgres
  • Versioning — re-POST with same id → v2, v3, …; `/view` is always latest, `/v1/view` is specific
  • Index with filters by type, author, title substring
  • Private by default — your team's network is the trust boundary

Read the deployment notes for a VPN-only setup or to put oauth2-proxy in front.

How is this different from a generic HTML pastebin?

Several free services let you upload an HTML file and get a public URL. They're fine for one-off shares. Pergam is built for the case where the AI agent makes the artifact, you want history, and your team needs to self-host behind a VPN.

FAQ

Who's this for?

Devs and small teams whose AI coding assistant generates a steady stream of HTML artifacts — PR review boards, status dashboards, evaluation reports, design specs with Mermaid — that today die in chat scrollback.

Why not just paste into Notion / Confluence?

Those strip CSS, break Mermaid, mangle dark themes, and require accounts. pergam keeps the artifact exactly as the AI rendered it.

Is the public service safe? What about the embedded HTML?

Grids are served from a sandboxed iframe on a different origin (api.pergam.dev) with a strict CSP that blocks everything except inline CSS and Mermaid from jsDelivr. Visitors are isolated. The 72-hour TTL also limits the blast radius of any abuse.

What about rate limits?

The public service caps each IP at 10 shares per hour and 60 reads per minute. Body size cap is 512 KB. Plenty for normal use, prevents one user from filling the worker quota.

Can I keep grids forever?

Yes — self-host. Public service is for ephemeral sharing. Self-host gives you a Postgres-backed, versioned, immutable history with no TTL.

What does the skill cost?

Nothing. It's a markdown file in this repo. The AI's API costs are separate (whatever you pay for your Claude Code / Cursor / etc).
