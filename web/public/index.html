Publish HTML grids your AI builds.

A tiny self-hostable HTTP service that stores HTML reports, dashboards, and design docs at stable, versioned URLs. The companion Claude Code skill turns a prompt into a shareable link.

Get started Star on GitHub

1 · Use with Claude Code

The intended flow. Drop the skill in ~/.claude/skills/ and ask Claude to "armá un grid HTML de X". The skill writes the HTML, posts here, and returns a URL.

git clone https://github.com/diesilveira/pergam
cp -r pergam/skills/post-html-grid ~/.claude/skills/

# point it at the free public service (72h TTL):
export PERGAM_URL=https://api.pergam.dev

# or at your own self-hosted instance:
export PERGAM_URL=http://localhost:1111

Then any future "publish this as a grid" in Claude Code lands here.

3 · Self-host

One docker compose up and you have your own pergam with Postgres, versioning, no TTL, and full ownership. Suitable for teams behind a VPN.

git clone https://github.com/diesilveira/pergam
cd pergam
docker compose up -d
open http://localhost:1111

Read the deployment notes for a VPN-only setup.

FAQ

Who's this for?

Devs and small teams whose AI coding assistant generates a steady stream of HTML artifacts — PR review boards, status dashboards, evaluation reports, design specs with Mermaid — that today die in chat scrollback.

Why not just paste into Notion / Confluence?

Those strip CSS, break Mermaid, mangle dark themes, and require accounts. pergam keeps the artifact exactly as the AI rendered it.

Is the public service safe? What about the embedded HTML?

Grids are served from a sandboxed iframe on a different origin (api.pergam.dev) with a strict CSP that blocks everything except inline CSS and Mermaid from jsDelivr. Visitors are isolated. The 72-hour TTL also limits the blast radius of any abuse.

What about rate limits?

The public service caps each IP at 10 shares per hour and 60 reads per minute. Body size cap is 512 KB. Plenty for normal use, prevents one user from filling the worker quota.

Can I keep grids forever?

Yes — self-host. Public service is for ephemeral sharing. Self-host gives you a Postgres-backed, versioned, immutable history with no TTL.

What does the skill cost?

Nothing. It's a markdown file in this repo. The AI's API costs are separate (whatever you pay for your Claude Code / Cursor / etc).
